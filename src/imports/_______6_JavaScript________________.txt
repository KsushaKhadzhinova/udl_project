API-ключи должны быть хранены в .env файле, но следует помнить, что это не защищает от утечек на клиенте.
API-ключи в клиентских приложениях всегда будут доступны конечному пользователю, но можно ограничить их использование:
Ограничьте использование API-ключей на сервере или используйте ограничения по IP.

В проекте можно также добавить механизм проверки безопасности, например:
Использование HTTPS для всех запросов.
Ограничение по методу HTTP, например, только GET для публичных данных.

JSON vs XML: What's the Difference?

XML (eXtensible Markup Language)
What is XML?

XML — расширяемый язык разметки для хранения и передачи структурированных данных, это стандартный формат для обмена данными, который раньше активно использовался, особенно в старых веб-сервисах и API.

Современные альтернативы и тренды
JSON: Для многих задач, особенно в веб-разработке и мобильных приложениях, JSON стал более популярной альтернативой из-за лаконичности и простой интеграции с JavaScript.

Языки разметки для данных: Для документов со сложной структурой (книги, техническая документация) часто используются специализированные языки на основе XML, такие как DocBook.

В итоге, хотя XML больше не является центральной технологией для фронтенда, он прочно занял нишу надёжного стандарта для структурированного межсистемного обмена данными, особенно в регулируемых отраслях и государственном управлении.

XML и его актуальность
XML остаётся важным стандартом, однако его роль и экосистема сильно изменились. Основная функция XML сместилась от описания веб-страниц к структурированному обмену данными между системами, в корпоративных и государственных цифровых сервисах.

XML до сих пор используется в некоторых областях, таких как:
Веб-сервисы на основе SOAP.
Некоторые специализированные форматы данных (например, в банковской сфере, здравоохранении, или в научных вычислениях).
Описание сложных документов, таких как RSS, RDF или WSDL.
Однако для веб-разработки JSON стал гораздо более предпочтительным форматом, так как он легче, быстрее и проще для работы в JavaScript.
Современные области применения XML
Сегодня XML — это язык-посредник для интеграции систем и основа для отраслевых стандартов, особенно в госсекторе.

Веб-сервисы (SOAP) и API: Хотя для публичных API чаще используется JSON, SOAP-сервисы на основе XML широко применяются в корпоративном секторе.

Конфигурационные файлы: Многие сложные системы (серверы, IDE) используют XML для хранения настроек.

Векторная графика (SVG): Формат SVG для описания масштабируемой графики полностью основан на XML.

Отраслевая стандартизация (на примере России):

Строительство и проектирование: С 2025 года вводятся обязательные XML-схемы для ведомостей объёмов работ (ВОР), пояснительных записок и актов испытаний.

Налоговая отчётность: С 1 апреля 2025 года универсальный передаточный документ (УПД) должен передаваться в ФНС в формате XML.

Кадастр и картография: Росреестр использует XML-схемы для формирования межевых планов.

Преимущества XML:
Гибкость: XML поддерживает сложные структуры данных и может быть использован для различных целей (например, в SOAP-сервисах).
Мощные механизмы обработки: XML имеет поддержу XPath, XSLT и других технологий для манипуляции с данными.
Технологии которые либо устарели, либо их роль сузилась до нишевых задач:

XSL (XSLT, XPath): XSLT (преобразование XML) и XPath (навигация по документу) активно используются. XSL-FO (для форматирования в PDF) сейчас применяется реже.

XLink и XPointer: Эти сложные спецификации для создания ссылок внутри XML не получили массового применения в веб-разработке.

XML Infoset и XML Base: Это скорее теоретические модели, на которые опираются другие стандарты, а не технологии для прямого использования разработчиками.

XSLT
XSLT остается действующим стандартом, но его роль сместилась из браузерной "веб-шаблонизации" в сторону мощного инструмента серверной обработки данных. Это не устаревшая технология, но сфера её применения изменилась и стала более специализированной.

Эволюция роли XSLT
Если в начале 2000-х XSLT часто использовался для преобразования XML в HTML прямо в браузере, то сегодня этот подход устарел. Современный веб строится на других технологиях (React, Vue.js, Angular), которые генерируют HTML динамически, а данные чаще передаются в формате JSON, а не XML.

Сейчас XSLT находит применение в более "фоновых" задачах, где требуется высокая надежность и стандартизация:

Преобразование данных между системами: Например, конвертация документов из одной XML-схемы в другую при интеграции корпоративных приложений.

Генерация отчетов: Создание структурированных документов (HTML, PDF, текстовых файлов) из XML-данных.

Обработка данных в профессиональных инструментах: Как показывают примеры из Microsoft Project и геоинформационной системы ArcGIS, XSLT встроен в сложные платформы для обработки метаданных, фильтрации и трансформации содержимого.

Прямое отображение XML в браузере устарело
Использование инструкции <?xml-stylesheet type="text/xsl" href="..."?> для отображения XML-документа через XSLT в браузере — устаревшая практика. Современные браузеры могут её поддерживать, но в веб-разработке она не используется. Преобразование почти всегда выполняется на стороне сервера с помощью XSLT-процессора, а пользователю отправляется готовый HTML, JSON или другой конечный формат.

Актуальные и неактуальные части стандарта XSL

XSLT (ядро): Актуален, но в версиях 2.0 (2007 г.) и 3.0 (2017 г.), которые значительно мощнее 1.0. Например, в XSLT 2.0 появилась встроенная группировка элементов, решающая сложную задачу из вашего материала более элегантно.

XPath: Критически важен и активно используется как в XSLT, так и в других технологиях (например, в XML Schema или при работе с DOM).

XSL-FO: Используется в нишевых задачах для генерации PDF из XML, но не является основной веб-технологией.

Современная интеграция с языками программирования
"XSLT-расширения" и интеграции с традиционными языками (Java, C#, Python) по-прежнему актуально. Сейчас это стандартный способ добавить в XSLT логику, которую сложно или невозможно реализовать на чистом XSLT. Например, для подключения к базе данных или вызова веб-сервиса в процессе трансформации.

XPath
XPath остается актуальным, так как он активно используется для работы с XML. Он часто применяется в системах, где нужно извлечь данные из XML или использовать их в XSLT.

Синтаксис XML
Фундаментальные правила синтаксиса, остаются неизменными и являются основой для работы с любыми современными XML-документами.

Корректность (well-formed): Обязательны один корневой элемент, правильная вложенность тегов, закрытие всех элементов, кавычки для атрибутов.

Декларация XML: Как правило, включает версию и кодировку (чаще всего UTF-8). Параметр standalone сейчас используется редко.

Синтаксис: Правила для комментариев, инструкций обработки (<? ... ?>), секций CDATA и escape-последовательностей (вроде &lt; для <) актуальны.

Пространства имён и валидация: переход к XML Schema
Раньше для описания структуры использовался DTD (Document Type Definition). Сейчас его почти полностью вытеснил XML Schema (XSD) — более мощный и гибкий язык, который сам является XML-документом.

Характеристика	DTD (Устаревший подход)	XML Schema (Современный стандарт)
Синтаксис	Свой собственный, не XML.	Основан на XML, с ним можно использовать те же инструменты.
Типы данных	Очень ограниченные (например, только строки).	Богатая система типов (числа, даты, логические значения, пользовательские типы).
Пространства имён	Не поддерживает.	Полная поддержка, что критично для объединения словарей из разных источников.
Расширяемость	Сложно расширять и повторно использовать.	Поддерживает наследование и переиспользование сложных типов.
Пространства имён XML (XML Namespaces) сегодня — это обязательный, а не рекомендуемый механизм. Они позволяют избежать конфликтов имён при использовании в одном документе тегов из разных словарей (например, из стандартов на строительство и геоданные).

JSON (JavaScript Object Notation)
Terminology: JSON(JavaScript Object Notation) – MIDAS Support

JSON — это легкий формат обмена данными, который используется для передачи данных между клиентом и сервером. Он стал стандартом для работы с данными в JavaScript, заменив более сложный формат XML.

JSON (JavaScript Object Notation) – это текстовый формат для хранения и обмена данными, основанный на синтаксисе JavaScript, который легко читается и обрабатывается как человеком, так и компьютером. Он основан на синтаксисе JavaScript-объектов, но используется независимо от языка программирования.

Преимущества JSON:
Читаемость: JSON легко читается и записывается человеком, а также хорошо интегрируется с JavaScript.
Меньший объем данных: JSON обычно легче, чем XML, что ускоряет процесс передачи данных.
Широкая поддержка: JSON поддерживается во всех современных языках программирования и широко используется в API.
Почему JSON так популярен?
Компактный и легко читаемый формат
Поддерживается всеми языками программирования
Легко преобразуется в объект в JavaScript
Используется в REST API, GraphQL, WebSockets и многих других технологиях

Современность:
Актуальность: JSON остается основным форматом для обмена данными между клиентом и сервером. Он используется в большинстве веб-API и является предпочтительным для современных приложений.
Что может его заменить: На данный момент JSON является наиболее распространенным форматом, и нет реальной замены. Однако в определенных сценариях используется XML или Protocol Buffers (особенно в системах с высокими требованиями к производительности), но они не так популярны в веб-разработке.

Асинхронное программирование - что это, понятие многопоточности

Асинхронность в JavaScript помогает улучшить производительность и отзывчивость приложений, позволяя выполнять другие операции, пока ожидается завершение долгих процессов, например, сетевых запросов или вычислений.

Промисы (Promises)
Promises — это улучшенная версия callback, которая позволяет работать с асинхронными операциями более удобно, поддерживает цепочку операций и обработку ошибок.

Promise - JavaScript | MDN

Промис представляет собой объект, который может быть в одном из трех состояний:

Ожидание (Pending) — промис еще не выполнен.
Выполнено (Resolved/fulfilled) — промис успешно завершен.
Отклонено (Rejected) — произошла ошибка при выполнении.

Почему асинхронное программирование важно?
Параллельные операции: Веб-приложения часто требуют выполнения операций, которые могут занять время, таких как запросы к серверу. Асинхронное программирование позволяет выполнять эти операции параллельно, не блокируя пользовательский интерфейс.
Производительность: Асинхронные операции позволяют уменьшить время ожидания и не блокируют основной поток выполнения, что улучшает отзывчивость приложения.
Управление ресурсами: Асинхронность позволяет эффективно использовать ресурсы, например, обрабатывать несколько запросов, не создавая излишней нагрузки на сервер.
Асинхронное программирование остается крайне важным в современной веб-разработке, особенно с развитием SPA (Single Page Applications) и использования различных API (например, RESTful и GraphQL). Оно также тесно связано с концепциями реального времени и обработкой данных с использованием WebSockets.

Множество популярных технологий, таких как Node.js (серверная среда для JavaScript), активно используют асинхронные операции, чтобы справляться с большим количеством параллельных запросов.

Fetch API – стандартный метод для работы с HTTP-запросами
Fetch API является современным и более удобным способом выполнения HTTP-запросов в JavaScript, в отличие от старого XMLHttpRequest. Это стандарт для асинхронных запросов в современных веб-приложениях.

Преимущества Fetch API:
Простота синтаксиса: Использует промисы, что делает код проще и читаемее.
Поддержка современных возможностей: Включает возможности для работы с кросс-доменными запросами, заголовками, и отправки данных.
Асинхронность с async/await: Легко используется вместе с конструкцией async/await, что упрощает работу с асинхронным кодом.
Fetch API позволяет получать данные с сервера с помощью fetch().

Что выбрать – Fetch или Axios?



Fetch хорош, если нужен стандартный API без зависимостей.
Axios удобнее, если планируется работать с API часто и нужны дополнительные возможности.
В современных проектах часто используют Axios из-за его удобства.
AJAX (XHR) – устаревший, но иногда используемый метод
AJAX (Asynchronous JavaScript and XML)  — это технология, позволяющая загружать данные асинхронно, не обновляя страницу. AJAX позволяет обмениваться данными с сервером без перезагрузки страницы. Технология популяризировала динамические веб-страницы и появилась еще в начале 2000-х годов, часто с использованием XMLHttpRequest для работы с API. Сейчас он заменен Fetch API и Axios.

Ajax Battle: XMLHttpRequest vs the Fetch API

Преимущества AJAX:
Асинхронность: Позволяет выполнять запросы в фоновом режиме, не блокируя пользовательский интерфейс.
Работа с XML и JSON: Хотя название AJAX включает XML, чаще всего используется JSON, так как он легче и удобнее для работы в JavaScript.
