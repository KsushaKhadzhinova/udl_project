## Ход работы

### 1. Анализ макета
На основе интерфейса в стиле VS Code были выделены основные функциональные зоны:

- панель инструментов;
- проводник;
- редактор кода;
- область предпросмотра;
- статус-бар.

### 2. Семантическая разметка
Вместо несемантичных контейнеров `<div>` использованы теги HTML5:

- `<header>` — шапка страницы;
- `<nav>` — навигация;
- `<aside>` — боковая панель;
- `<main>` — основной контент;
- `<section>` — логические блоки;
- `<figure>` — иллюстрация или диаграмма;
- `<footer>` — нижняя часть страницы.

### 3. Методология БЭМ
Для именования классов применён стандарт БЭМ.  
Примеры:

- `ide-header__title`;
- `ide-btn--save`.

### 4. Доступность (A11y)
Для повышения удобства использования были внедрены следующие решения:

- добавлен `skip-link` для быстрого перехода к редактору;
- использованы атрибуты `aria-label` для кнопок-иконок;
- соблюдена логическая иерархия заголовков `h1-h3`.

### 5. Микроразметка Schema.org
В проект были добавлены следующие типы данных:

- `SoftwareApplication` — описание самого редактора;
- `Organization` — информация о разработчике;
- `BreadcrumbList` — навигационная цепочка пути к файлу;
- `FAQPage` — раздел с вопросами и ответами.

## Ответы на контрольные вопросы

### Какие семантические теги использованы?
Использованы теги `<header>`, `<nav>`, `<aside>`, `<main>`, `<section>`, `<figure>`, `<footer>`.

### Как иерархия заголовков влияет на доступность?
Она помогает пользователям со скринридерами быстро ориентироваться в структуре страницы и переходить между разделами.

### Зачем нужна микроразметка?
Микроразметка помогает поисковым системам лучше понимать содержание страницы и формировать расширенные сниппеты в выдаче.
